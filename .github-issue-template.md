# 🎨 Redesign GitHub Pages with Modern Web Stack

## Overview
Transform the current basic HTML GitHub Pages site into a modern, professional website using Astro, Tailwind CSS, and shadcn/ui components to better showcase Prompt Injectionator and serve different user cohorts effectively.

## Current State
- Basic HTML site in `docs/index.html`
- Minimal styling and basic layout
- Single-page approach with limited navigation
- GitHub Pages serving from `docs` folder on main branch

## Goals
- **Professional Appearance**: Modern, clean design that builds trust
- **User-Centric Navigation**: Clear paths for 4 distinct user cohorts:
  - **Curious n00bs** (Level 0): Complete newcomers to prompt injection concepts
  - **Beginners** (Level 100): Know about it but need guidance
  - **Intermediates** (Level 200): Ready to use the tool
  - **Advanced** (Level 300): Want to develop and extend

## Technical Approach
- **Static Site Generator**: Astro (perfect for static sites with great performance)
- **Styling**: Tailwind CSS for modern, responsive design
- **Components**: shadcn/ui for professional UI components
- **Build Process**: `/website` source → build to `/docs` for GitHub Pages
- **Compatibility**: Maintain current GitHub Pages deployment from docs folder

## Site Structure
```
/                    # Landing page with clear cohort navigation
├── /curious         # Level 0: "What are prompt injections?"
├── /getting-started # Level 100: Installation and first steps
├── /guide          # Level 200: Comprehensive usage guide
├── /advanced       # Level 300: Architecture, APIs, contributing
├── /docs           # Technical documentation
└── /examples       # Live demos and examples
```
